---
#### Remarks

\<p>
    That is, builds the operations that will take the database from the state left in by
    this migration so that it returns to the state that it was in before this migration was applied.
\</p>
\<p>
    This method must be overridden in each class that inherits from \<xref href="Microsoft.EntityFrameworkCore.Migrations.Migration" data-throw-if-not-resolved="false">\</xref> if
    both 'up' and 'down' migrations are to be supported. If it is not overridden, then calling it
    will throw and it will not be possible to migrate in the 'down' direction.
\</p>
\<p>
    See  for more information and examples.
\</p>
